# Unified Regular-Gain Gate Final Results

This record fixes the current reportable incremental method over reproduced TurboQuant.

## Method

Use the same prompt-only gate at both 2.5-bit and 3.5-bit:

- Default path: TurboQuant MSE.
- Active path: switch both K and V to `regular_gain_mse`.
- Activate only when the prompt is not code-like, `prompt_tokens > 0`, `passage_count <= 12`, and `question_marks <= 20`.
- `lcc` and `repobench-p` are treated as code-completion tasks and always stay on TurboQuant MSE.

This is one unified rule and one code path for both bit budgets.
